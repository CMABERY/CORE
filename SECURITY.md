# Security Policy

This repository currently contains a minimal, dependency-light prototype.

## Reporting a vulnerability
- Do **not** open a public issue with exploit details.
- Use your internal security channel or contact the repository owners directly.

## Scope notes
- This project may execute code generation / patch application logic.
- Treat any workspace used for demos as untrusted and disposable.

## Response
We will acknowledge receipt, assess severity, and publish fixes via tagged commits.
